FAQ

Common questions from editors.

Does it upload my footage? +
No. Everything runs locally on your machine using Whisper via Apple's MLX framework. Your video files never leave your drive. No account, no internet connection required for processing.
Does it work offline? +
Yes, after the initial model download. CutSmith runs entirely offline — no API calls, no cloud dependency during processing.
Does it support bilingual subtitles? +
Yes. CutSmith is designed for Chinese/English mixed-language audio and can generate separate EN and ZH subtitle tracks from the same source file. It handles code-switching audio better than a standard single-language transcription pass, though accuracy varies with audio quality.
Does it work with long-form interviews and documentaries? +
Yes — long-form content is the primary use case CutSmith is built for. It processes files in segments to manage memory, and maintains timecode continuity across the full duration.
Does it work with Premiere Pro? +
Yes. Export an SRT file and import it directly into Premiere Pro via the Captions panel. Output is also tested with Final Cut Pro, DaVinci Resolve, and CapCut. Minor formatting differences may appear depending on NLE version.
Can I edit subtitles before exporting? +
Yes. The subtitle editor lets you edit text, adjust timecodes, and review segments before export. Smart Repair handles the most common cleanup automatically, but full manual control is available throughout.
Does it support Chinese and English mixed audio? +
Yes — mixed ZH/EN audio is a core design target. CutSmith is built to handle rapid code-switching better than a single-language model pass. Results depend on audio clarity and speaker separation; very noisy recordings may still require manual review.
ABOUT THIS TOOL

CutSmith is built from real post-production workflows — long-form interviews, variety production, and cross-NLE delivery. The first tool grew out of a recurring problem: getting clean, editable bilingual subtitles out of mixed-language rushes without spending two hours on cleanup. The second is solving another: moving a rough cut out of CapCut and into a professional finishing workflow without rebuilding the edit from scratch. These are indie tools in active development, not a platform. Feedback from working editors shapes every release.
